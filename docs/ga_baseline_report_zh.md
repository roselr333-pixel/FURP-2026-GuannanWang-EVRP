# GA 基线报告 — 官方 Solomon VRPTW（100 客户标准集）

> - 作者：Guannan Wang（学生）
> - 日期：2026-07-17
> - 范围：自己实现的一套遗传算法（Genetic Algorithm），作为 OR-Tools 之外的"外部基线"，在 `benchmark_official_solomon.py` 所用的同一批 56 个官方 Solomon 实例上评测。
> - 说明：这是我写的一个经典元启发式，主要当作一次小练习，让自己对"启发式是怎么搭出来的"有个基本实感。它**不是**论文复现，也**不打算**和 OR-Tools 比高低。

## 1. 为什么加 GA 基线

我 Week-5 的主求解器是 OR-Tools（引导式局部搜索），并已对照官方 56 实例的 BKS（平均 gap 7.2%）。OR-Tools 效果不错，但只用它，我对"路径启发式到底是怎么搭出来的"其实没什么实感。所以我自己试着写了一个遗传算法，在**同一批**实例上跑，至少保证对比用的是同样的数据、同样的 BKS。

## 2. GA 是怎么搭的

| 组件 | 选择 | 原因 |
|---|---|---|
| 编码 | 客户排列（先排路线） | 最简单、最经典的 GA 表示 |
| 解码 | Solomon I1 插入法 | 能得到**可行**路线集（容量+时间窗都满足），且车辆数一般较少 |
| 适应度 | 总距离 + 车队规模双向惩罚 + `force_fleet` 拆分 | 尽量让 GA 和 BKS 用**同一车队规模**比距离（否则宽窗实例会用更少车、显得 artificially 更好） |
| 选择 | 锦标赛（规模 4）+ 精英保留（6 个直接复制） | 保住最优解，同时保留探索 |
| 交叉 | 顺序交叉 OX | 标准排列算子 |
| 变异 | 交换 + 反转，概率 0.3 | 局部扰动 |
| 局部搜索 | 路线内 2-opt + 路线间 relocate | 打磨每个个体 |
| 热启动 | Clarke–Wright savings 种子 | 一个还算可行的开局；对聚类型（C 类）实例效果不错 |
| 预算 | 种群 40，每实例 8 秒，固定随机种子 20260717 | 让 56 实例整轮控制在约 11 分钟内 |

## 3. 结果

**56/56 全部求解成功。GA 相对 BKS 的平均 gap = 36.2%。**（对照：OR-Tools 在同一批上平均 gap = 7.2%。）

各族平均 gap（GA vs BKS）：

| 族 | 数量 | 平均 gap | 最小 | 最大 |
|---|---|---|---|---|
| C1 | 9 | 29.1% | 0.2% | 90.6% |
| C2 | 8 | 31.5% | 0.4% | 61.9% |
| R1 | 12 | 31.7% | 15.2% | 45.9% |
| R2 | 11 | 41.0% | 32.6% | 57.5% |
| RC1 | 8 | 29.9% | 19.5% | 39.3% |
| RC2 | 8 | 55.1% | 35.0% | 68.9% |

## 4. 这些数字说明什么

- **GA 在每一个实例上都弱于 OR-Tools**（三方对比里 `ga_minus_ortools_gap` 全为正）。这很正常：OR-Tools 用的是工业级引导式局部搜索，而我这只是基础的教科书式元启发式。
- **聚类型实例方差极大。** C101 / C105 / C106 做到距 BKS 仅 0.2%（接近 OR-Tools），但 C103 飙到 90.6%。savings 种子帮了简单的聚类情况，却牺牲了别处的稳定性——这是真实短板。
- **宽时间窗（R2 / RC2）最难**（41% / 55%）。只有 8 秒预算、局部搜索又基础，GA 在这些更松散、更组合化的实例上几乎收敛不了。
- 附带观察：很多 R/RC 实例上 OR-Tools 自己用的车辆数**少于** BKS，却付出更多距离——那是我在 checkpoint 里讲过"车辆固定成本建模"导致的，不是 GA 的问题。

## 5. 局限（诚实）

- 只跑了一遍、单一随机种子、每实例仅 8 秒——GA 结果本就有随机性，且预算很小。
- 没做参数调优；局部搜索是基础的。
- 它只是通用 VRPTW 启发式，**没有**适配我项目真正关心的 E-VRPTW 充电约束。

## 6. 结论与下一步

GA 基线只是一次初步尝试。它明显弱于 OR-Tools，但让我在看待自己的结果时有了一个粗糙的参照。它不是论文复现，所以对我来说真正有意思的问题不是"我能不能用玩具 GA 打败 OR-Tools"，而是"已发表的 E-VRPTW 方法（Schneider 2014、Murray & Chu 2015）表现如何、我能不能复现它们？"——这正是我拿到这两篇 PDF 后就要开始的论文复现路线。
